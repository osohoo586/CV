latin2cyrillic-convertor/ NLP R&D

Mongolian Latin → Cyrillic Transliterator

Албан бус латин үсгээр бичсэн монгол текстийг (Facebook, чат) зөв кирилл рүү хөрвүүлдэг hybrid NLP pipeline. 4 шатлалт: Trie dict → bigram/trigram LM → byte-level ByT5 seq2seq → vowel-harmony checker. 3M synthetic pair-тэй corpus, Apple M1 (MPS) дээр локалаар инференц.

Hybrid NLP pipeline that converts informal Latin-script Mongolian (Facebook, chat) into correct Cyrillic. Four-stage: Trie dict → bigram/trigram LM → byte-level ByT5 seq2seq → vowel-harmony checker. Trained on 3M synthetic pairs, runs locally on Apple M1 (MPS).

ByT5 N-gram LM Trie Vowel Harmony seq2seq on-device 3M synthetic
Дэлгэрэнгүй More details

Шат 1 — Trie dict lookup: Хамгийн түгээмэл үгсийг O(k) хурдтай dictionary-ээр хайдаг (neural cost-гүй fast path).

Шат 2 — Bigram/Trigram LM: Trie-д олдоогүй эсвэл олон утгат үгсийг N-gram language model-аар heuristic disambiguation хийдэг.

Шат 3 — ByT5 seq2seq: OOV (out-of-vocab) болон complex case-уудыг byte-level ByT5-small transformer-аар хөрвүүлдэг. Tokenization-ийн алдаа байхгүй.

Шат 4 — Vowel harmony: Монгол хэлний агглютинатив суффиксийн vowel harmony дүрмийг deterministic rule-аар шалгаж засдаг.

Өгөгдөл: 3M synthetic parallel pair бүхий corpus үүсгэсэн. Apple M1 (MPS) GPU дээр локалаар train + inference хийдэг тул API хамаарал, latency байхгүй.

Stage 1 — Trie dict lookup: O(k) lookup for the most common words (a no-neural-cost fast path).

Stage 2 — Bigram/Trigram LM: N-gram LM heuristically disambiguates words missed by the trie or with multiple valid mappings.

Stage 3 — ByT5 seq2seq: Out-of-vocab and complex cases are converted by a byte-level ByT5-small transformer — no tokenization artefacts.

Stage 4 — Vowel harmony: A deterministic rule layer checks and corrects vowel-harmony violations in agglutinative suffixes.

Data: a 3M synthetic parallel-pair corpus was generated. Training and inference run locally on Apple M1 (MPS) — no API dependency, no network latency.

Odoo Documentation RAG Chatbot

Англи Odoo баримтуудыг уншиж монгол хариулт өгдөг RAG чатбот. Section-based chunking, idempotent indexing (нэг л удаа embedding), top-K retrieval + LLM хариулт. .txt / .md / .rst / .html дэмжинэ.

A RAG chatbot that ingests English Odoo docs and answers in Mongolian. Section-based chunking, idempotent indexing, top-K retrieval + LLM generation. Supports .txt / .md / .rst / .html.

FastAPI Qdrant text-embedding-3-large PostgreSQL cross-lingual
Дэлгэрэнгүй More details

Section-based chunking: Документыг тэмдэглэгээний section-аар нь смайсат байдлаар хуваадаг — Token boundary биш semantic boundary.

Idempotent indexing: Файлын checksum хадгалж, өөрчлөгдөөгүй документыг дахин embed хийдэггүй — restart-д хурдан, OpenAI quota хэмнэлттэй.

Retrieval + Generation: Top-K semantic search (K=5) → асуудалд хамаатай context-ыг LLM (GPT-3.5/4o-mini) руу дамжуулна → монгол хариулт буцаана. Уг pipeline нь .txt / .md / .rst / .html форматыг дэмжинэ.

History: Бүх асуулт хариултыг PostgreSQL дээр хадгалж, цаашид retrain эсвэл analytics-д ашиглах боломжтой.

Section-based chunking: Documents are split along marked sections — splitting on semantic boundaries, not token boundaries.

Idempotent indexing: File checksums are stored so unchanged documents aren't re-embedded — fast restarts and lower OpenAI quota usage.

Retrieval + generation: top-K semantic search (K=5) → relevant context is sent to the LLM (GPT-3.5/4o-mini) → Mongolian answer returned. The pipeline supports .txt / .md / .rst / .html.

History: Every Q&A is stored in PostgreSQL, available for retraining or analytics.

6 audio features Mel-spectrogram · MFCC · ZCR · Chroma · Spectral Contrast · Tonnetz

02 · Augmentation

03 · Training

5 ангилал → belly pain · burping · discomfort · hungry · tired 5 classes → belly pain · burping · discomfort · hungry · tired

During my university studies I experimented with several machine-learning methods — traffic-light recognition, object detection from images, and classifying infant cries by cause. The most significant of these was the infant cry-classification project.

Deep Learning / CNN: Extracted Mel-spectrogram, MFCC, ZCR, Chroma, Spectral Contrast and Tonnetz features, fed them into a Convolutional Neural Network (CNN), and classified infant cries into 5 categories — belly pain, burping, discomfort, hungry, and tired.

Data Augmentation: Used tempo/pitch shifting and noise injection to generate 300+ samples per class, improving model generalization.

PyTorch & Torchaudio: PyTorch for deep-learning model training, validation, and hyperparameter tuning. Torchaudio for converting audio files into tensors, producing spectrograms, and running fast GPU-accelerated processing.
